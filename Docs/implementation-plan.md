# Implementation Plan: AI-Powered Discovery Engine (Part 1)

This document outlines the phase-wise implementation plan to build the AI-Powered Discovery Engine for Swiggy Instamart, based on the requirements in `problemStatement.md` and the system design in `architecture.md`.

## Phase 1: Data Ingestion (The Scrapers)
We will build the data extraction layer to gather unstructured feedback.
- [NEW] `scripts/ingest_playstore.py`: Script using `google-play-scraper` to fetch recent Swiggy Instamart reviews.
- [NEW] `scripts/ingest_reddit.py`: Script using `praw` to fetch discussions from target subreddits.
- [NEW] `data/raw_reviews.csv`: Local storage for the raw, combined data.
- [NEW] `requirements.txt`: We will add the initial dependencies (`google-play-scraper`, `praw`, `pandas`).

## Phase 2: Data Processing & Normalization
We will clean the raw data so the AI models don't get confused by noise.
- [NEW] `scripts/process_data.py`: A script using `pandas` to filter out short reviews (< 8 words), remove emojis, and standardize the text.
- [NEW] `data/clean_reviews.csv`: The output of the processing script, ready for clustering.

## Phase 3: Semantic Clustering (Theme Identification)
We will use machine learning to automatically group the clean reviews into themes (e.g., "Spoilage Fears", "Search Habits").
- [NEW] `scripts/cluster_themes.py`: A script using `sentence-transformers` to generate embeddings, and `umap-learn`/`hdbscan` to cluster the vectors.
- [NEW] `data/clustered_themes.json`: Stores the representative quotes for each identified cluster.
- *Dependency Update*: Add `sentence-transformers`, `umap-learn`, and `hdbscan` to `requirements.txt`.

## Phase 4: AI Reasoning (The Groq LLM)
We will feed the clustered themes into the LLM to answer the 8 core strategic questions.
- [NEW] `scripts/generate_insights.py`: A script that reads `clustered_themes.json` and sends a highly specific prompt to the Groq API (using Llama 3).
- [NEW] `data/final_insights.json`: The structured answers generated by the LLM, complete with citations linking back to the raw quotes.
- *Dependency Update*: Add `groq` to `requirements.txt`.

## Phase 5: Delivery UI (Render Deployment)
We will build a simple UI to demonstrate the workflow and present the insights.
- [NEW] `app/main.py`: A lightweight FastAPI backend (or Streamlit) to serve the insights.
- [NEW] `Dockerfile`: Configuration files to easily deploy the application to Render.

## Verification Plan
- **Phase 1 & 2**: Run the ingest scripts manually and verify that `clean_reviews.csv` contains high-quality, English-only text.
- **Phase 3**: Verify `clustered_themes.json` correctly groups semantic themes without duplicate garbage data.
- **Phase 4**: Run the Groq query and verify the output rigidly conforms to the required markdown table structure in `final_insights.json`.
- **Phase 5**: Run the UI locally, verify it displays the data correctly, and then deploy to Render.
